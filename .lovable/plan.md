# Фаза 1 — Починка блокера (пустая страница на /finance)

## Что делаем

### 1. `src/routes/__root.tsx`
- Убрать `ssr: false` из root route (дублируется с `_authenticated/route.tsx` и ломает `MatchInnerImpl`).
- В `ErrorComponent` защититься от `undefined`: `error?.message ?? "Неизвестная ошибка"`, плюс fallback на кнопку «На главную» если `reset` недоступен.

### 2. `src/routes/_authenticated/route.tsx`
- Добавить `errorComponent` (показывает сообщение + кнопки «Повторить» / «На главную» / «Выйти»).
- Добавить `notFoundComponent` (404 внутри защищённой зоны со ссылкой на `/`).
- Оставить `ssr: false` и `beforeLoad` как есть.

### 3. Проверка в браузере
Пройти сценарий:
1. `/` — главная грузится
2. `/finance` — больше не белый экран
3. `/students` — список
4. `/schedule` — календарь
5. Добавить студента — `owner_id` проставляется триггером, запись видна
6. Settings → Выход → редирект на `/auth`
7. Переход на `/finance` без сессии → редирект на `/auth`

Если что-то падает — читаем console + network, фиксим точечно.
