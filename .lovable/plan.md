
## Что меняем

### 1. Статусы посещения (обновить ATT_STATUS)
- ✅ **Был** — считается в шкалу уроков
- ❌ **Отсутствовал** — тоже считается в шкалу уроков (как пройденный, ученик платит)
- 📎 **Уваж. причина** — НЕ считается в основную шкалу, копится в отдельной шкале «Шансы на перенос» (макс 3)
- 🔄 **Перенос учителем** — НЕ считается в шкалу уроков, создаёт долг учителя (учитель должен ученику урок). Отображается в карточке ученика как «Должен Х уроков».

### 2. Основная шкала «Цикл уроков» (12)
Заполняется по `present + absent` (а не только `present` как сейчас). При достижении 12/12:
- Автоматически создаётся новая запись в `finance` со статусом «Не оплачено» (сумма по умолчанию из `user_settings.default_lesson_price * 12`, валюта из настроек).
- Старая оплата НЕ трогается (остаётся в истории как оплачено).
- Toast + push «Цикл завершён — добавлен новый долг на оплату».

### 3. Новая шкала «Шансы на перенос» (0/3)
- Считает количество записей со статусом `excused` у ученика (не удалённых).
- При попытке записать 4-й `excused` — toast.error «Лимит уваж. причин (3) исчерпан» и блокировка записи.
- Отображается отдельным баджем/прогрессом под основной шкалой.

### 4. Долг учителя (перенос учителем)
- Считает количество `rescheduled_by_teacher` минус число «возмещённых» уроков.
- Возмещение: новый статус возмещения не нужен — будем считать, что любой следующий `present` сверх запланированных компенсирует долг. **Уточнение для упрощения:** просто показываем счётчик «Должен N уроков» = count(rescheduled_by_teacher) без автоматического списания; вычитание делается вручную (галочкой на записи). 

  Альтернатива: добавить boolean колонку `compensated` в `attendance` для записей со статусом `rescheduled_by_teacher`, и кнопку «Отметить как возмещённый».

### 5. Журнал долгов (общий обзор)
На странице **Главная** (index) или новой секции в `StudentRoom` показать список «Я должен»: ученик → N уроков переносов.
